# 企业级页面规范

## 页面结构

- 页头只放页面标题、关键元信息和当前页面主操作。
- 主操作放在页头右侧，顺序为：主流程动作、下一步入口、辅助动作、导出/下载、危险操作。
- 上传、筛选、批量处理等上下文工具放在独立工具条或表单区，不挤占页头标题区。
- 非任务必需的说明面板不放在业务列表页底部；列表页优先展示数据、筛选和操作。

## 按钮规范

- 普通按钮使用 `.btn`，主动作使用 `.btn.primary`，危险动作使用 `.btn.danger`。
- 按钮文字不换行；行内操作按钮固定宽度，避免表格操作列抖动。
- 表格操作列只保留 1 个查看入口和 1 个下一步主操作入口，避免多按钮堆叠。
- 禁用按钮必须使用不可操作样式，不使用可点击链接模拟禁用。

## 表单与上传

- 文件上传控件必须放在独立导入条或表单字段内，带清晰标签。
- 文件上传按钮与“保存/发布/删除”等模板管理动作分组展示。
- 表单字段使用稳定的网格布局，标签左/上对齐，避免控件挤压。

## 数据展示

- 标签、状态、关键词必须允许换行，不允许挤压成竖排。
- 长文本字段使用自然换行或分段列表，不能逐字符渲染。
- 卡片只承载一组完整业务信息；不嵌套卡片，不把导航说明当作业务面板。

## 常见页面布局

- 列表页：工具条 + 指标概览 + 数据表，不放流程说明/待办提醒面板。
- 详情页：页头动作 + 关键指标 + 业务内容卡片 + 必要审计/后台任务。
- 配置页：左侧列表 + 右侧详情，详情页头放保存/发布/删除，导入文档单独成工具条。
- 审核页：页头放确认/下一步/重生成/导出，内容区按“列表-预览-交付”组织。
